Neural Network Architectures for Urine Analysis

Comparing MLP, CNN, and Transformer approaches to medical classification

0:00 / 0:00

Multi-Layer Perceptron (MLP)

Best for: Tabular biomarker data (numeric readings)

  • Input: 5 numeric features (pH, Protein, Glucose, Specific Gravity, Leukocytes)
  • Architecture: Fully-connected layers where every neuron connects to every neuron in the next layer
  • Strengths: Simple, fast to train, effective for structured tabular data
  • Limitation: Cannot capture spatial or sequential patterns in data
0:00 / 0:00

Convolutional Neural Network (CNN)

Best for: Image-based analysis (urine test strip color patterns)

  • Input: Urine test strip image with colored reagent pads
  • Architecture: Sliding filters detect local patterns, pooling reduces dimensions, then dense layers classify
  • Strengths: Excels at detecting spatial patterns, color gradients, and visual features
  • Limitation: Requires image data; less effective for pure numeric inputs

Transformer (Self-Attention)

Best for: Sequential/longitudinal patient data over time

  • Input: Time-series of biomarker readings across multiple days
  • Architecture: Self-attention lets each time point "look at" all others to find relevant patterns
  • Strengths: Captures long-range dependencies, finds correlations across time points
  • Limitation: Requires more data and compute; overkill for simple tabular inputs
